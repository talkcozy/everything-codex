# Everything Codex 贡献指南

感谢你对 Everything Codex 项目的兴趣！

## 如何贡献

### 报告问题

1. 检查是否已有类似 issue
2. 使用 issue 模板创建新 issue
3. 提供详细的复现步骤
4. 包含环境信息（OS、Codex 版本等）

### 提交代码

1. Fork 本仓库
2. 创建功能分支 (`git checkout -b feature/amazing-feature`)
3. 提交更改 (`git commit -m 'Add amazing feature'`)
4. 推送到分支 (`git push origin feature/amazing-feature`)
5. 创建 Pull Request

### 提交规范

```
<type>: <subject>

<body>

<footer>
```

类型：
- `feat`: 新功能
- `fix`: Bug 修复
- `docs`: 文档更新
- `style`: 格式调整
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具

### 代码风格

- 遵循项目现有的代码风格
- 确保所有测试通过
- 更新相关文档

## 开发流程

1. **讨论**：在 issue 或 discussion 中讨论重大变更
2. **开发**：在本地分支开发
3. **测试**：确保功能正常
4. **文档**：更新相关文档
5. **提交**：创建 PR 并等待审查

## 贡献领域

### 技能 (Skills)

- 添加新的开发工作流技能
- 改进现有技能文档
- 翻译技能文档

### 规则 (Rules)

- 添加新语言的编码规则
- 更新现有规则
- 修复规则中的错误

### 智能体 (Agents)

- 创建新的智能体配置
- 优化智能体指令
- 分享使用案例

### 文档

- 改进 README
- 添加使用示例
- 翻译文档

## 行为准则

- 接受建设性批评
- 关注社区最佳利益
