# 调试 / Debug

系统性地诊断和修复问题。

## 使用方式

```bash
# 调试特定错误
codex /debug "API returns 500 error on login"

# 调试性能问题
codex /debug "page loads slowly"

# 调试测试失败
codex /debug "test UserService.test.js is failing"
```

## 调试流程

### 1. 问题确认
- 明确问题症状
- 收集错误信息
- 确定重现步骤
- 识别影响范围

### 2. 信息收集
- 查看日志
- 检查配置
- 分析环境
- 审查变更

### 3. 假设生成
- 提出可能原因
- 按概率排序
- 设计验证方法

### 4. 问题定位
- 缩小问题范围
- 检查相关代码
- 验证假设
- 定位根因

### 5. 修复验证
- 实施修复
- 验证修复
- 回归测试
- 文档记录

## 输出格式

```markdown

### 问题症状
- **现象**: [描述]
- **错误信息**: [错误]
- **重现步骤**:
  1. [步骤]
  2. [步骤]

### 诊断过程

#### 假设 1: [假设描述]
- **验证**: [如何验证]
- **结果**: [验证结果]

#### 假设 2: [假设描述]
...

### 根本原因
[确定的根因]

### 修复方案
```[代码/配置]
```

### 验证
- [ ] 问题已修复
- [ ] 无回归问题
- [ ] 测试通过

### 预防措施
[如何避免类似问题]
```
