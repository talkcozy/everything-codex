# Security Review 技能

受 ECC 项目启发，专为 Codex 优化的安全审查工作流。

## 描述

Security Review 技能提供系统化的安全检查流程，帮助识别代码中的安全漏洞和风险。

## 使用场景

- 代码提交前的安全检查
- 安全审计
- 敏感功能开发审查
- 依赖项安全评估
- 配置文件安全审查

## 安全审查清单

### 输入验证

- [ ] 所有用户输入都经过验证
- [ ] 使用白名单而非黑名单
- [ ] 验证数据类型、长度、格式和范围
- [ ] 文件上传验证 MIME 类型和内容
- [ ] 路径遍历防护

### 认证与授权

- [ ] 使用强密码策略
- [ ] 实施多因素认证
- [ ] 会话管理安全
- [ ] 权限最小化原则
- [ ] API 密钥不存储在代码中

### 数据保护

- [ ] 敏感数据加密存储
- [ ] 传输中使用 HTTPS/TLS
- [ ] 密码使用强哈希算法（bcrypt, Argon2）
- [ ] 不记录敏感信息
- [ ] 正确处理机密信息

### 常见漏洞防护

- [ ] SQL 注入（使用参数化查询）
- [ ] XSS 攻击（输出编码）
- [ ] CSRF 防护
- [ ] 命令注入
- [ ] SSRF 防护
- [ ] 不安全的反序列化

### 依赖安全

- [ ] 使用 `npm audit` / `pip audit` 检查漏洞
- [ ] 定期更新依赖
- [ ] 移除未使用的依赖
- [ ] 锁定依赖版本

## 工作流程

### Step 1: 范围界定

```
1. 确定审查范围（文件/模块/功能）
2. 识别敏感数据和关键路径
3. 了解威胁模型
```

### Step 2: 自动化扫描

```
1. 运行静态安全扫描工具
2. 检查依赖漏洞
3. 审查安全警告
```

### Step 3: 手动审查

```
1. 检查输入验证点
2. 审查认证/授权逻辑
3. 检查敏感数据处理
4. 审查配置和密钥管理
5. 检查错误处理和日志
```

### Step 4: 报告和修复

```
1. 分类发现的问题（Critical/High/Medium/Low）
2. 提供修复建议
3. 跟踪修复进度
4. 验证修复效果
```

## Codex 指令模板

```
请对以下代码进行安全审查：

审查范围：[文件/模块路径]

重点关注：
- [ ] 输入验证
- [ ] 认证授权
- [ ] 数据保护
- [ ] 依赖安全

请提供：
1. 发现的漏洞列表（按严重程度排序）
2. 每个漏洞的具体位置和建议修复
3. 一般性的安全改进建议
```

## 严重级别定义

| 级别 | 描述 | 响应时间 |
|------|------|----------|
| 🔴 Critical | 可被利用获取系统控制权或敏感数据 | 立即修复 |
| 🟠 High | 可能导致数据泄露或严重功能损坏 | 24小时内 |
| 🟡 Medium | 可能造成有限影响的问题 | 1周内 |
| 🟢 Low | 轻微问题或深度防御建议 | 下次迭代 |
