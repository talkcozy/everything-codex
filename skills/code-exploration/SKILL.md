# Code Exploration 技能

受 ECC 项目启发，专为 Codex 优化的代码探索工作流。

## 描述

Code Exploration 技能提供系统化的代码库分析方法，帮助快速理解复杂项目结构和代码逻辑。

## 使用场景

- 新项目 onboarding
- 代码库健康检查
- 重构前的影响分析
- 代码审查准备
- 技术债务评估

## 探索策略

### 1. 鸟瞰视图

首先了解项目整体结构：

```
1. 阅读项目文档（README、CONTRIBUTING、ARCHITECTURE）
2. 分析目录结构
3. 识别技术栈和主要依赖
4. 了解构建和部署流程
5. 理解测试结构
```

### 2. 关键路径追踪

识别和追踪核心数据流：

```
1. 找到入口点（main、index、app）
2. 追踪请求/数据流
3. 识别核心业务逻辑
4. 理解状态管理
5. 识别外部依赖和集成点
```

### 3. 依赖分析

理解模块间关系：

```
1. 绘制模块依赖图
2. 识别循环依赖
3. 找到高耦合区域
4. 识别核心抽象和接口
5. 分析配置和常量定义
```

## 分析维度

### 代码组织

- **模块化**: 职责分离是否清晰
- **命名规范**: 是否一致且语义化
- **文件组织**: 是否遵循既定结构
- **复杂度**: 函数/类大小是否合理

### 质量指标

- **测试覆盖**: 核心逻辑是否有测试
- **文档**: 关键部分是否有文档
- **错误处理**: 是否有适当的错误处理
- **代码异味**: 重复代码、过长函数等

### 技术债务

- **TODO/FIXME**: 标记的问题数量
- **弃用代码**: 是否有死代码
- **依赖陈旧**: 依赖是否过时
- **架构偏离**: 是否有偏离架构的代码


```
项目路径：[项目路径]

请重点关注：
1. 整体架构和技术栈
2. 核心业务逻辑和数据流
3. 代码质量和测试覆盖
4. 潜在的技术债务
5. 需要关注的安全或性能问题
```

## 工具推荐

- **结构分析**: `tree`, `cloc`
- **依赖分析**: `depcheck`, `dependency-cruiser`
- **复杂度分析**: `complexity-report`, `radon`
- **测试覆盖**: `coverage.py`, `nyc`, `go test -cover`
