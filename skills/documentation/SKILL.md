# Documentation 技能

系统化的技术文档编写指南。

## 文档类型

### 1. README.md
项目入口文档，包含：
- 项目简介
- 安装说明
- 快速开始
- 主要功能
- 贡献指南
- 许可证

### 2. API 文档
```markdown
## [API 名称]

### 请求
```http
METHOD /path
Content-Type: application/json
Authorization: Bearer {token}
```

### 参数
| 名称 | 类型 | 必填 | 描述 |
|------|------|------|------|
| name | string | 是 | 名称 |

### 响应
```json
{
  "code": 200,
  "data": {}
}
```
```

### 3. 架构文档
- 系统架构图
- 数据流
- 技术选型
- 部署架构

### 4. 开发指南
- 开发环境搭建
- 代码规范
- 提交流程
- 调试技巧

## 文档原则

### 清晰性
- 使用简洁语言
- 避免术语堆砌
- 提供具体示例
- 使用列表和表格

### 完整性
- 覆盖所有功能
- 包含边界情况
- 提供故障排除
- 更新变更日志

### 一致性
- 统一术语
- 统一格式
- 统一风格
- 定期审查

## 文档模板

### 功能文档
```markdown
## [功能名称]

### 概述
[功能简介]

### 使用场景
- [场景 1]
- [场景 2]

### 使用方法
```[代码]
```

### 参数说明
| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|

### 注意事项
- [注意点]

### 示例
[具体示例]
```

### ADR (架构决策记录)
```markdown
# ADR-XXX: [决策标题]

## 状态
[提议/已接受/已弃用]

## 背景
[决策背景]

## 决策
[具体决策]

## 后果
### 正面
- [好处]

### 负面
- [代价]

## 替代方案
- [方案 A] - [为什么不选]
```
